# Chimera
Your personal file headed monster.

## Origin Story
In Greek mythology, a *chimera* was a monster composed of many animal parts. Likewise, the purpose of this program is to combine many source files into a singular file. In my research, I found a number of programs that would bundle code into an executable; however, my use case is a little different. I don't need an executable, I need an intelligent file merger. In my world, I have a library of useful python snippets that I want to optionally combine. I want to specify a list of files and merge them into one to be used elsewhere. 

## Why?
This tool will enable teams to maintain a library of "useful code", which can then be curated, tailored, and vultured for specific projects.

## Use Case
As an AI Engineer building AI Agents, each build looks different, but is composed of similar parts. These parts can be organized into logical groupings and maintained in their own files. This pattern puts emphases on good code review practices. With the library of files, an engineer can choose 80% of them to be included in a build. Notably, this also works when an engineer wants to include something later in a project's life cycle. All they have to do is select the files they want, merge them, and copy and paste to their development environment.
